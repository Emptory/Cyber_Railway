# Cyber Railway - 数据传输服务架构

## 项目概述
- **队友模型端**: Python模型，提供数据接口
- **数据传输服务**: 你负责开发，获取模型数据并上传到阿里云
- **阿里云服务端**: 接收和存储数据

## 架构图
```
┌─────────────────┐    HTTP/API    ┌─────────────────┐    HTTP/HTTPS    ┌─────────────────┐
│   队友模型服务   │  ────────────> │  数据传输服务    │  ─────────────> │   阿里云服务器   │
│  (提供数据接口)  │                │   (你开发)      │                 │  (接收存储数据)  │
└─────────────────┘                └─────────────────┘                 └─────────────────┘
```

## 数据传输服务职责
1. **数据获取**: 从队友模型接口拉取数据
2. **数据处理**: 数据格式转换、验证、清洗
3. **数据上传**: 可靠地传输到阿里云服务器
4. **错误处理**: 重试机制、异常监控
5. **日志记录**: 完整的操作日志

## 技术选型
- **语言**: Python 3.8+
- **HTTP客户端**: requests + aiohttp
- **任务调度**: APScheduler
- **配置管理**: YAML + python-dotenv
- **日志**: loguru
- **重试机制**: tenacity
